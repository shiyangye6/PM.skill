# Product Manager Archetypes

当用户不需要点名某位产品经理时，优先使用这些抽象原型。

## 用户价值型 PM

核心判断：

- 谁的什么问题被解决？
- 用户当前替代方案是什么？
- 价值是否足以让用户改变行为？

常用产物：

- 用户画像
- Job-to-be-done
- 替代方案分析
- 用户价值假设

## 发现型 PM

核心判断：

- 现在最大的未知是什么？
- 需要访谈、原型、数据还是实验？
- 先验证问题，还是先验证方案？

常用产物：

- 假设清单
- 访谈提纲
- 机会树
- 实验计划

## 体验型 PM

核心判断：

- 用户第一次使用是否顺畅？
- 核心路径是否足够短？
- 产品是否有清晰的默认行为？
- 哪些细节会造成不信任？

常用产物：

- 用户流程
- 状态设计
- 空状态和错误状态
- 可用性检查清单

## 增长型 PM

核心判断：

- 第一批用户从哪里来？
- 激活时刻是什么？
- 留存和传播靠什么？
- 商业化是否破坏体验？

常用产物：

- 漏斗
- 增长实验
- 冷启动计划
- 指标树

## 执行型 PM

核心判断：

- MVP 是否能在当前资源下完成？
- 需求边界是否明确？
- 开发是否知道先做什么？
- 验收标准是否可测试？

常用产物：

- PRD
- 任务拆分
- 里程碑
- 验收标准

## 技术协作型 PM

核心判断：

- 需求是否能转成清晰的数据模型和状态机？
- 技术约束会如何影响体验？
- 哪些边界情况必须提前处理？
- 哪些接口需要先定义？

常用产物：

- DevDoc
- 数据模型
- API 草案
- 状态流转
- 测试计划
